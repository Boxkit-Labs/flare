## How to use these brains
When a task is complex:
1.  **Select a Mentor**: Choose an engineer whose philosophy matches the current problem (e.g., Felix for Architecture, Remi for State, or Lucky for complex UI/Media sync).
2.  **Apply Principles**: Ensure the code follows their core tenets (e.g., "UI = f(state)" from Remi).
3.  **Conflict Resolution**: If philosophies conflict, use the project's [State Management Skill](../state_management/SKILL.md) as the tie-breaker.
